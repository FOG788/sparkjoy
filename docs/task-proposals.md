# コードベース概要と改善タスク提案

最終更新: 2026-03-11

## コードベース概要

- 構成は `index.html`（UI と説明書）、`app.js`（ロジック本体）、`styles.css`（見た目）、`tests/regression.test.mjs`（回帰テスト）を中心にした静的サイトです。
- 保存・コピー・クリア、入力速度指標（CPM/WPM）、しきい値判定、砂時計ウィジェット、効果音制御、ローカル永続化（`localStorage` + cookie）が `app.js` に集約されています。
- テストは「既定値の整合」「ハイスコアの比較基準」「説明書文言の方向性」を正規表現で検証する最小セットです。

## 提案タスク（4件）

### 1) 入力ミス修正タスク
**対象:** `README.md`, `index.html`

- タスク: README の「エディタ」表記を UI と同じ「エディター」へ統一し、軽微な入力ゆれを修正する。
- 背景: README では「エディタ」、画面上では「SparkJoy Editor / エディター版」となっており、用語統一の観点で小さなノイズになっている。

### 2) バグ修正タスク
**対象:** `app.js`

- タスク: `Persistence.getCookie` の cookie 分割ロジックを `split('; ')` 固定から、空白有無に依存しない実装（例: `split(';').map(trim)`）へ変更する。
- 背景: 現実の `document.cookie` は `;` の後の空白が必ずしも一定ではないため、現在の実装だと cookie 読み出しに失敗する可能性がある。

### 3) コメント/ドキュメント矛盾修正タスク
**対象:** `index.html`（説明書）

- タスク: 「WPM」の説明を「平均WPM」と明記するか、UI ラベルを `平均WPM` に変更して表示名と説明の粒度を合わせる。
- 背景: 現在は表示ラベルが `WPM`、説明文は「セッション全体の平均WPM」となっており、瞬間値と誤解される余地がある。

### 4) テスト改善タスク
**対象:** `tests/regression.test.mjs`

- タスク: 既存の正規表現ベース検証に加えて、`hourglassDuration` / `hourglassOpacity` / `autoReset` の UI 既定値と `app.js` の `load*` 既定値一致テストを追加する。
- 背景: しきい値系の整合は既に検証されている一方、他の永続化項目は同じ事故パターン（UI とロジック既定値の乖離）を防げていない。
