# SparkJoy リポジトリ全体レビュー（オミット候補）

最終更新: 2026-03-09

## 結論（優先度つき）

### P0（先に削る/直すべき）
1. **存在しない `manifest.json` 参照を除去**
   - `index.html` は `./manifest.json` を参照しているが、リポジトリ内に実ファイルがない。
   - PWA運用しないなら `<link rel="manifest">` は削除対象。

2. **README の GitHub Actions / Pages 記述を整理**
   - README では `.github/workflows/deploy-pages.yml` 等が「追加済み」とあるが、現状リポジトリに workflow ファイルが存在しない。
   - 現状に合わせて削除/更新しないと運用手順として誤誘導になる。

### P1（機能整理の効果が高い）
3. **未使用関数 `isIOS` を削除**
   - `isIOS()` が定義のみで参照されていないため、保守ノイズになっている。

4. **未使用エフェクト関数 `spawnCrackRing` を削除**
   - `spawnCrackRing()` は定義されているが呼び出しがなく、現在は `spawnSingleImpactAround()` に一本化されている。

5. **設定項目の削減（特に音響の微調整）**
   - 設定タブに「リアル度 / リバーブ / ジャム確率 / 黄赤しきい値 / ウォームアップ」まで並んでおり、初見ユーザーには調整負荷が高い。
   - MVP としては、`音量` と `エフェクトON/OFF` 以外を隠し設定に寄せる余地が大きい。

### P2（容量・体験の最適化）
6. **効果音サンプル数の縮小（4→1〜2）**
   - `sounds/` に複数 MP3 を同梱し、起動後にまとめてデコードしている。
   - 静的配信の初期体験を軽くするなら、サンプル点数を絞るか、合成音のみ運用に寄せる選択が可能。

7. **「説明書」タブの分離検討**
   - 画面内に長文の説明書を持つ構成は、エディタUIとしては情報量過多になりやすい。
   - README か別ヘルプページへ切り出すと、編集画面をよりミニマルにできる。

## 補足（判断基準）
- 本レビューは「壊れている箇所の修正」ではなく、**機能削減・要素削減の候補出し**にフォーカス。
- 削除判断は次の優先順を推奨:
  1) 実体のない参照（manifest / workflow記述）
  2) 未使用コード
  3) UXの学習コストを上げる設定
  4) 容量コストの高い演出資産
