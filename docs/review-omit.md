# SparkJoy リポジトリ全体レビュー（オミット候補）

最終更新: 2026-03-10

## 結論（優先度つき）

### P0（先に直すべき）
1. **ウォームアップ既定値の不一致を修正**
   - UI 既定値は `10s` だが、復元ロジック側の既定値が `12` になっていた。
   - 初期表示と設定復元時で値がズレるため、`10` で統一する。

2. **説明書のしきい値説明を実装仕様に合わせる**
   - 実装は「入力速度がしきい値以上になると黄/赤ギミック」を発火する設計。
   - 説明文を「速度低下」基準で記述していると誤解を招くため修正する。

### P1（機能整理の効果が高い）
3. **未使用関数 `isIOS` を削除検討**
   - `isIOS()` が定義のみで参照されていないため、保守ノイズになっている可能性がある。

4. **未使用エフェクト関数 `spawnCrackRing` を削除検討**
   - `spawnCrackRing()` は定義されているが呼び出しがなく、現在は `spawnSingleImpactAround()` に一本化されている。

5. **設定項目の段階表示（詳細設定の折りたたみ）を検討**
   - 「リアル度 / リバーブ / ジャム確率 / 黄赤しきい値 / ウォームアップ」まで常時表示され、初見ユーザーには調整負荷が高い。
   - MVP としては、`音量` と `エフェクトON/OFF` を主表示、残りは詳細設定に寄せる余地がある。

### P2（容量・体験の最適化）
6. **効果音サンプル数の縮小（4→1〜2）を検討**
   - `sounds/` に複数 MP3 を同梱し、起動後にまとめてデコードしている。
   - 静的配信の初期体験を軽くするなら、サンプル点数を絞るか、合成音のみ運用に寄せる選択が可能。

7. **「説明書」タブの分離検討**
   - 画面内に長文の説明書を持つ構成は、エディタUIとしては情報量過多になりやすい。
   - README か別ヘルプページへ切り出すと、編集画面をよりミニマルにできる。

## 補足（判断基準）
- 本レビューは「壊れている箇所の修正」ではなく、**機能削減・要素削減の候補出し**にフォーカス。
- 削除判断は次の優先順を推奨:
  1) 現行挙動と説明の不一致
  2) 未使用コード
  3) UXの学習コストを上げる設定
  4) 容量コストの高い演出資産
